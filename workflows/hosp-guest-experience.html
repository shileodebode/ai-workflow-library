In-Stay Request Pipeline · Room 214
RECEIVE
💬WhatsApp Guest Msg
WhatsApp Business API webhook captures guest messages 24/7. Tunde's message from Room 214 is received and parsed in milliseconds.
→
🤖GPT-4o Intent Parse
GPT-4o classifies Tunde's message into 3 parallel requests: housekeeping (pillows), maintenance (AC noise), and concierge (suya spot). Routes each independently.
ROUTE
🛏️Housekeeping Slack
Posts to #housekeeping Slack channel: "Room 214 — 2 extra pillows requested. Guest: Tunde Adeleke. ETA needed." Housekeeper confirms within 3 mins.
→
🔧Maintenance Ticket
Creates a maintenance ticket in the PMS: "Room 214 AC making noise — priority: medium." Assigns nearest available engineer. Tracks until closed.
→
🗺️Concierge KB Query
Queries the hotel's curated knowledge base + Google Maps for suya restaurants near Victoria Island. Returns top 3 rated options with hours and distance.
RESPOND
✍️GPT-4o Compose Reply
GPT-4o combines all 3 responses into one professional, friendly WhatsApp message — ETAs, engineer alert, and restaurant pick — no info overload.
→
📤Unified WA Reply
Single WhatsApp reply sent to Tunde addressing all 3 requests simultaneously. Response time: under 90 seconds from first message.
→
📊Request Dashboard
All requests logged in Google Sheets — timestamp, room, type, resolution time, status. Front desk and managers see live status without calling anyone.
